# Product Vision

## Definition

Morrow is a self-hosted personal AI agent for people who want serious capability without surrendering control of their data or accepting a rough developer-only experience.

It should perform the same broad classes of work as Hermes Agent while feeling calmer, clearer, more customizable, more reliable, and easier to trust.

Morrow is not an operating system. It is not merely a dashboard for another agent. It is the complete agent product: interface, orchestration, memory, tools, automation, integrations, and execution history.

## Product promise

**Private intelligence, built around you.**

Morrow should be:

- Easy enough to use without reading documentation
- Powerful enough for developers, founders, and technical operators
- Customizable without forcing every user to become an administrator
- Transparent about what it is doing, what it knows, and what leaves the machine
- Dependable during long-running and failure-prone tasks

## Primary users

### Builder

Uses Morrow for coding, research, testing, deployment preparation, and managing long-running technical work.

### Operator

Uses Morrow for recurring workflows, monitoring, reports, planning, communications, and connected services.

### Privacy-focused user

Needs local storage, local-model support, visible external data flow, and strict project boundaries.

## Experience model

Morrow exposes three levels of control:

1. **Presets** — polished configurations that work immediately.
2. **Studio** — visual control over agents, models, memory, tools, autonomy, interface, and privacy.
3. **Developer mode** — files, APIs, plugins, skills, MCP, custom providers, and event hooks.

Complexity should appear only when the user asks for it.

## Differentiators

- Persistent named agents instead of temporary anonymous delegation
- Structured, inspectable, scoped memory
- Resumable tasks with checkpoints and recovery
- Visual plans and execution evidence
- Clear privacy and provider disclosures
- Reversible changes and project history
- Deep customization across behavior and interface
- Repeatable public benchmarks against alternatives

## Non-goals for the first release

- Enterprise administration
- Public plugin marketplace
- Multi-tenant hosted service
- Autonomous financial transactions
- Broad computer control without scoped permissions
- Feature volume without reliability evidence
