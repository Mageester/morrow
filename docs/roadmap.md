# Roadmap

The roadmap prioritizes reliability and proof over broad feature count.

## Stage 0 — Foundation

- Repository standards and CI
- Product and architecture documents
- Hermes capability research
- Threat and privacy model
- Design-system foundations
- Protocol contracts

## Stage 1 — Reliable vertical slice

- Web shell
- Local project storage
- Conversation and task creation
- Visible plans
- One model provider and one local-provider path
- One safe filesystem or terminal tool
- Event timeline
- Checkpoint persistence
- Restart recovery
- Privacy and cost inspector

## Stage 2 — Useful personal agent

- File and terminal tools
- Browser research
- Model routing and fallbacks
- Structured project memory
- Diff and rollback
- Skills and MCP foundations
- Scheduling
- Desktop packaging

## Stage 3 — Persistent agent teams

- Named agents
- Per-agent roles, memory, tools, permissions, and budgets
- Task graph
- Parallel workspaces
- Independent review and verification
- Stuck-task detection

## Stage 4 — Hermes compatibility and parity

- Supported configuration import
- Memory and session import
- Skills compatibility
- Messaging adapters
- Broader tool coverage
- Published parity suite

## Stage 5 — Superiority evidence

- Public benchmark runner
- Reliability and recovery tests
- Privacy verification
- Usability comparisons
- Performance and cost reporting
- Documented release criteria
