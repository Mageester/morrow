Section 1

What the First Demo Attempt Found

Assembling the original evidence package surfaced a real bug: the exported report's ## Final Answer contained the same planning sentence repeated 12 times.

"Now I have full context. Let me apply all the changes." "Now I have full context. Let me apply all the changes." "Now I have full context. Let me apply all the changes." ... (12 occurrences, concatenated from every ReAct turn)

ROOT CAUSENeither the orchestrator's ReAct loop nor the CLI's terminal state reducer tracked turn boundaries — every turn's narration was folded into one ever-growing message.

The defect found2 / 12
Section 2

The Fix — PR #36

  • Orchestrator emits assistant.turn_started / assistant.turn_completed per ReAct turn, with a stable turnId and a final flag set only on the turn that produced no further tool calls.
  • CLI's state reducer mirrors turn identity explicitly — a delta only appends to its own open turn; an unmatched turn is dropped with a visible notice, never silently merged.
  • selectCanonicalFinalAnswer() in output-report.ts is the single source of truth used by /output, /output full, /export, and the one-shot --json path alike.
  • Intermediate turns demoted to a bounded ## Intermediate Activity section; recovery events get ## Recovery Summary.

MERGED46 new/updated tests, including a regression fixture built from the real corrupted task. Full validation clean twice. CI green.
