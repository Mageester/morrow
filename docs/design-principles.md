# Design Principles

Morrow should feel calm, deliberate, fast, and coherent. “Apple-like” means clarity and attention to detail, not imitation of Apple branding or interface assets.

## Principles

### One clear next action

Each screen should make the primary action obvious without hiding important state.

### Progressive disclosure

Basic controls remain simple. Advanced model, permission, memory, workflow, and interface controls appear when requested.

### Activity without log noise

Show plans, progress, agents, files, costs, and approvals in human language. Raw logs remain available in an inspector.

### Contextual controls

Use a contextual inspector rather than permanent settings panels across every screen.

### Reversible by default

Destructive actions, file edits, memory changes, and automation updates should support undo or restoration wherever practical.

### Dense when useful

Coding, operations, and activity views may be information-dense, but spacing and hierarchy must remain disciplined.

### Honest states

Distinguish clearly between planned, running, blocked, failed, verified, and complete. Never use “done” when verification failed.

### Accessible motion and contrast

Animation should explain state, not decorate it. Respect reduced-motion settings and keyboard navigation.

## Core navigation

- Today
- Conversations
- Projects
- Agents
- Automations

Secondary surfaces belong in contextual navigation or settings rather than expanding the main sidebar indefinitely.
