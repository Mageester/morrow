# Hermes Parity and Superiority Matrix

Morrow must not claim to replace or exceed Hermes without measurable evidence.

## Status values

- `Research` — current Hermes behavior still needs to be documented.
- `Planned` — Morrow requirements are defined but not implemented.
- `Partial` — usable subset exists.
- `Parity` — required baseline behavior passes comparison tests.
- `Exceeded` — Morrow passes a documented higher standard.

## Comparison rules

- Use the same task, model class, machine, network conditions, and time limit.
- Record versions and configuration.
- Separate capability presence from quality.
- Publish failed tests as well as successful ones.
- Avoid cherry-picked demonstrations.
- Do not mark `Exceeded` without a reproducible benchmark or usability study.
